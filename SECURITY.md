# Security Policy

## Supported versions

Security fixes are applied to the latest released version.

| Version | Supported |
| --- | --- |
| `0.0.x` | Yes |
| Older versions | No |

## Reporting a vulnerability

Please report suspected vulnerabilities privately through a GitHub Security Advisory when the repository is hosted on GitHub. If private advisories are unavailable, email [dominik.hladik@seznam.cz](mailto:dominik.hladik@seznam.cz).

Do not open a public issue before a fix or coordinated disclosure is available. Include the affected version, impact, reproduction steps, and a minimal sanitized input when possible. Do not send proprietary game databases or unrelated personal data.

Relevant reports include out-of-bounds parsing, uncontrolled resource consumption, malicious metadata behavior, unsafe serialized output, and dependency vulnerabilities that affect this package. Unsupported FIFA platforms or ordinary malformed-file errors without a security impact should use the normal issue tracker.

Reports will be investigated on a best-effort basis. Please allow time to reproduce, patch, test, and release a correction before public disclosure.
